# Polyglot Bridge —— 多语者大模型翻译助手

## 项目简介

Polyglot Bridge 是一款基于 Electron + React 构建的便携免安装桌面翻译工具。它通过集成多种大语言模型（LLM）API，为多语言学习者和工作者提供灵活、可定制的翻译与语言处理环境。

### 核心特性

- **便携免安装**：基于 Electron 打包，单文件或绿色文件夹即可运行
- **多模型支持**：集成 OpenAI、Anthropic、DeepSeek、Google Gemini、Moonshot 等主流 LLM 提供商
- **项目化管理**：以"项目"为单位隔离配置与工作上下文，方便切换不同翻译场景
- **流式翻译**：支持流式输出，实时查看翻译结果
- **划词翻译**：通过全局快捷键 `Alt+T` 从剪贴板捕获文本并翻译
- **多模式运行**：支持翻译全文、解释单词、总结段落、自定义指令四种模式
- **Markdown 渲染**：翻译结果支持 Markdown 格式渲染

## 技术栈

| 层级 | 技术 | 用途 |
|------|------|------|

| 路由 | React Router 7 | 页面导航（Memory Router） |

## 架构设计

### 整体架构

```
┌─────────────────────────────────────────────────────┐
│                    Electron 主进程                    │
│  ┌──────────┐ ┌──────────────┐ ┌──────────────────┐ │
│  │ storage  │ │ llm-adapter  │ │  word-capture    │ │
│  │ (SQLite) │ │ (HTTP请求)    │ │ (全局快捷键)     │ │
│  └────┬─────┘ └──────┬───────┘ └────────┬─────────┘ │
│       │              │                   │           │
│       └──────────────┼───────────────────┘           │
│                      │ IPC (invoke/on/send)          │
├──────────────────────┼───────────────────────────────┤
│                      │ preload.ts (contextBridge)    │
├──────────────────────┼───────────────────────────────┤
│                React 渲染进程                         │
│  ┌──────────────────────────────────────────────┐   │
│  │              ipc-client.ts                    │   │
│  │         (统一的bridge API封装)                 │   │
│  └──────────────────┬───────────────────────────┘   │
│                     │                                │
│  ┌──────────┐ ┌─────┴─────┐ ┌───────────────────┐  │
│  │ Zustand  │ │  Pages    │ │   Components      │  │
│  │ Stores   │ │ (6个页面)  │ │ (Markdown等)      │  │
│  └──────────┘ └───────────┘ └───────────────────┘  │
└─────────────────────────────────────────────────────┘
```

### 数据流

1. **配置流程**：用户在 SetupPage 填写配置 → useAppStore 更新状态 → bridge.saveConfig() → IPC → storage.ts 写入 SQLite
2. **翻译流程**：用户输入文本 → RuntimePage 调用 bridge.translateStream() → IPC → llm-adapter.ts 发起 HTTP 请求 → 流式返回结果 → 渲染 Markdown
3. **划词流程**：用户按 Alt+T → word-capture.ts 读取剪贴板 → IPC 广播 → RuntimePage 接收 → 自动翻译 → FloatingResult 显示

### LLM 适配器设计

[`llm-adapter.ts`](electron/modules/llm-adapter.ts) 通过统一接口适配不同提供商：

| 提供商 | API端点 | 认证方式 |
|--------|---------|----------|
| OpenAI | api.openai.com/v1/chat/completions | Bearer Token |
| Anthropic | api.anthropic.com/v1/messages | x-api-key Header |
| DeepSeek | api.deepseek.com/v1/chat/completions | Bearer Token |
| Gemini | generativelanguage.googleapis.com/v1beta | Bearer Token |
| Moonshot | api.moonshot.cn/v1/chat/completions | Bearer Token |

每个提供商的请求体格式和流式解析逻辑都有差异，适配器内部统一处理后暴露简洁接口。
