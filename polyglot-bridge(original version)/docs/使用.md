# Polyglot Bridge —— 使用指南

## 一、环境准备

### 1.1 系统要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11、macOS 12+、Linux (Arch/Debian) |
| Node.js | >= 18.0.0 |
| npm | >= 9.0.0 |

### 1.2 安装依赖

```bash
# 进入项目根目录
cd "0/polyglot-bridge(original version)"

# 安装所有依赖（首次运行必须执行）
npm install
```

> **解析**：`npm install` 会读取 [`package.json`](0/polyglot-bridge(original version)/package.json) 中的 `dependencies` 和 `devDependencies`，下载并安装到 `node_modules/` 目录。主要依赖包括：
> - `react` / `react-dom`：UI框架
> - `electron`：桌面应用框架
> - `better-sqlite3`：本地数据库
> - `zustand`：状态管理
> - `tailwindcss`：CSS框架
> - `vite` + `vite-plugin-electron`：构建工具

## 二、启动开发环境

### 2.1 启动命令

```bash
npm run dev
```

> **解析**：该命令执行 [`package.json`](0/polyglot-bridge(original version)/package.json:6) 中定义的脚本：
> ```json
> "scripts": {
>   "dev": "vite"
> }
> ```
> Vite 会同时启动：
> 1. **前端开发服务器**（默认端口 5173）：提供 React 热更新
> 2. **Electron 主进程**：通过 [`vite-plugin-electron`](0/polyglot-bridge(original version)/vite.config.ts:9) 自动编译并启动桌面窗口

### 2.2 开发流程

启动后会自动打开桌面窗口，应用流程如下：

```
欢迎页 (3秒) → 主页(三选项) → 初步设置 / 项目管理 / 应用设置
```

## 三、核心功能使用

### 3.1 初步设置（配置API）

**路径**：主页 → ⚙️ 初步设置

**操作步骤**：
1. 选择模型提供商（如 OpenAI、DeepSeek）
2. 输入对应的 API Key
3. 点击「测试连接」验证密钥是否有效
4. 可选：选择具体模型、设置源语言和目标语言
5. 点击「保存并返回主页」

**关键代码解析** —— [`SetupPage.tsx`](0/polyglot-bridge(original version)/src/features/setup/SetupPage.tsx:1)：

```tsx
// 测试连接：调用bridge API验证密钥
const handleTest = async () => {
  setTesting(true);
  const result = await bridge.testConnection({
    apiProvider, apiKey,
    modelTag: modelTag || currentProvider?.models[0] || ''
  });
  setTestResult(result);
  setTesting(false);
};
```

> **解析**：`bridge.testConnection()` 通过 IPC 调用主进程的 [`llm-adapter.ts`](0/polyglot-bridge(original version)/electron/modules/llm-adapter.ts:147)，向对应提供商发送一个简单的 "Hello" 请求来验证连接。

```tsx
// 保存配置：更新Zustand状态 + 持久化到SQLite
const handleSave = async () => {
  const newConfig = { apiProvider, apiKey, modelTag, sourceLang, targetLang };
  setConfig(newConfig);    // 更新内存状态
  await saveConfig();      // 通过IPC写入数据库
  navigate('/home');       // 跳转回主页
};
```

> **解析**：配置通过两步保存——先更新 Zustand store（内存），再通过 `bridge.saveConfig()` → IPC → [`storage.ts`](0/polyglot-bridge(original version)/electron/modules/storage.ts:24) 写入 SQLite 数据库。

### 3.2 项目管理

**路径**：主页 → 📁 项目管理

**操作步骤**：
1. 点击「+ 新建项目」，输入项目名称
2. 项目会自动继承初步设置中的配置
3. 点击项目查看详情，可编辑名称和备注
4. 点击「▶ 启动运行」打开独立运行窗口

**关键代码解析** —— [`ProjectsPage.tsx`](0/polyglot-bridge(original version)/src/features/project-manager/ProjectsPage.tsx:1)：

```tsx
// 创建项目：使用当前全局配置作为默认值
const handleCreate = async () => {
  await createProject({
    name: newName.trim(),
    apiProvider: config.apiProvider,  // 继承全局配置
    apiKey: config.apiKey,
    modelTag: config.modelTag,
    sourceLang: config.sourceLang,
    targetLang: config.targetLang,
    note: '',
  });
};
```

> **解析**：新建项目时自动填入全局配置，用户可在项目详情中单独修改，不影响全局设置。这实现了需求文档中的「项目数据继承」功能。

```tsx
// 启动运行窗口：通过IPC创建独立的Electron窗口
const handleStart = (id: string) => {
  bridge.startRuntime(id);
};
```

> **解析**：`bridge.startRuntime()` → IPC → [`project-runner.ts`](0/polyglot-bridge(original version)/electron/modules/project-runner.ts:7) 创建一个新的 BrowserWindow，加载 `/runtime/{projectId}` 路由。每个项目有独立窗口，互不干扰。

### 3.3 运行窗口（翻译操作）

**路径**：项目管理 → 选择项目 → ▶ 启动运行

**功能说明**：

| 功能 | 说明 |
|------|------|
| 翻译全文 | 将输入文本翻译为目标语言 |
| 解释单词 | 详细解释单词的含义、词性、用法 |
| 总结段落 | 对输入文本进行摘要 |
| 自定义指令 | 输入任意提示词，自由控制LLM行为 |
| 划词翻译 | 启用后，按 Alt+T 从剪贴板捕获文本翻译 |
| 复制结果 | 一键复制翻译结果到剪贴板 |
| 重新生成 | 重新调用LLM获取新的翻译结果 |

**关键代码解析** —— [`RuntimePage.tsx`](0/polyglot-bridge(original version)/src/features/runtime/RuntimePage.tsx:1)：

```tsx
// 四种运行模式，每种对应不同的系统提示词
const MODES = [
  { value: 'translate', label: '翻译全文',
    prompt: 'You are a professional translator...' },
  { value: 'explain', label: '解释单词',
    prompt: 'You are a language teacher...' },
  { value: 'summary', label: '总结段落',
    prompt: 'You are a summarization assistant...' },
  { value: 'custom', label: '自定义指令', prompt: '' },
];
```

> **解析**：每种模式预设了不同的系统提示词（System Prompt），指导LLM以不同方式处理输入文本。自定义模式允许用户自由输入提示词。

```tsx
// 流式翻译：实时接收LLM返回的token
const doTranslate = useCallback((text: string, isFloating = false) => {
  const prompt = getPrompt();
  let full = '';

  bridge.translateStream(config, text, prompt, (chunk: string) => {
    if (chunk === '[DONE]') {
      setStreaming(false);           // 流结束
    } else if (chunk.startsWith('[ERROR]')) {
      setOutput(`❌ 翻译失败`);      // 错误处理
    } else {
      full += chunk;                 // 拼接token
      setOutput(full);               // 实时更新UI
    }
  });
}, [config, getPrompt]);
```

> **解析**：`bridge.translateStream()` 通过 IPC 发送翻译请求到主进程。主进程的 [`llm-adapter.ts`](0/polyglot-bridge(original version)/electron/modules/llm-adapter.ts:107) 发起 HTTP 流式请求，逐个 chunk 通过 IPC 回传给渲染进程，实现打字机效果。

```tsx
// 划词翻译监听：接收主进程广播的剪贴板文本
useEffect(() => {
  const cleanup = bridge.onCapturedText((text: string) => {
    doTranslate(text, true);  // isFloating=true 显示悬浮窗
  });
  return cleanup;  // 组件卸载时清理监听器
}, [doTranslate]);
```

> **解析**：当用户在任意窗口按 `Alt+T`，[`word-capture.ts`](0/polyglot-bridge(original version)/electron/modules/word-capture.ts:8) 读取剪贴板内容，通过 IPC 广播给所有窗口。RuntimePage 监听该事件，自动触发翻译并显示在悬浮窗中。

### 3.4 划词翻译

**操作步骤**：
1. 在运行窗口点击「✂️ 启用划词」
2. 在任意应用中选中文字并复制（Ctrl+C）
3. 按 `Alt+T` 触发翻译
4. 悬浮窗显示翻译结果，可固定、复制、关闭

**关键代码解析** —— [`word-capture.ts`](0/polyglot-bridge(original version)/electron/modules/word-capture.ts:1)：

```tsx
function startCapture() {
  if (captureActive) return;
  captureActive = true;

  // 注册全局快捷键 Alt+T
  globalShortcut.register('Alt+T', () => {
    const text = clipboard.readText().trim();  // 读取剪贴板
    if (text && text !== lastText) {           // 去重
      lastText = text;
      // 广播给所有窗口
      BrowserWindow.getAllWindows().forEach(win => {
        if (!win.isDestroyed()) {
          win.webContents.send('word-capture:on-text-captured', text);
        }
      });
    }
  });
}
```

> **解析**：使用 Electron 的 `globalShortcut` API 注册系统级快捷键。按下后读取剪贴板内容（用户需先 Ctrl+C 复制文本），去重后广播给所有窗口。

## 六、IPC 通信架构详解

应用采用 Electron 标准的 IPC 通信模式：

```
渲染进程                    主进程
┌──────────┐  invoke   ┌──────────────┐
│ bridge   │ ────────→ │ ipcMain      │
│ API      │           │ .handle()    │
│          │ ←──────── │              │
└──────────┘  result   └──────────────┘

┌──────────┐  send     ┌──────────────┐
│ bridge   │ ────────→ │ ipcMain      │
│ (事件)   │           │ .on()        │
│          │ ←──────── │ sender.send  │
└──────────┘  回调      └──────────────┘
```

**IPC 通道列表**：

| 通道 | 方向 | 类型 | 用途 |
|------|------|------|------|
| `storage:get-config` | 渲染→主 | handle | 获取全局配置 |
| `storage:save-config` | 渲染→主 | handle | 保存全局配置 |
| `storage:project:list` | 渲染→主 | handle | 获取项目列表 |
| `storage:project:create` | 渲染→主 | handle | 创建项目 |
| `storage:project:update` | 渲染→主 | handle | 更新项目 |
| `storage:project:delete` | 渲染→主 | handle | 删除项目 |
| `llm:test-connection` | 渲染→主 | handle | 测试API连接 |
| `llm:translate` | 渲染→主 | handle | 非流式翻译 |
| `llm:translate-stream` | 渲染→主 | on/send | 流式翻译 |
| `word-capture:start` | 渲染→主 | send | 启动划词 |
| `word-capture:stop` | 渲染→主 | send | 停止划词 |
| `word-capture:on-text-captured` | 主→渲染 | send | 广播捕获文本 |
| `project:start-runtime` | 渲染→主 | send | 启动运行窗口 |

## 七、常见问题

### Q1: 启动后白屏？
确保已执行 `npm install`，且 Node.js 版本 >= 18。

### Q2: 测试连接失败？
检查 API Key 是否正确，网络是否能访问对应提供商的 API 端点。

### Q3: 划词翻译不工作？
确保已在运行窗口点击「启用划词」，且先用 Ctrl+C 复制文本再按 Alt+T。

### Q4: 如何添加新的 LLM 提供商？
编辑 [`llm-adapter.ts`](0/polyglot-bridge(original version)/electron/modules/llm-adapter.ts:8) 中的 `PROVIDERS` 对象，添加新的端点和默认模型即可。
